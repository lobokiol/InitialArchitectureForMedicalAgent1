# 会话叙事逻辑摘要（可跨项目复用）

## 一、会话在解决什么（元目标）

不是单讲某个仓库，而是建立一套 **可复述、可面试、可落地** 的叙述方式：

- 项目值不值得、做到哪一档（评测 / 准上线 / 真上线）
- 架构怎么拆（双通道、AI vs 传统、一期 vs 二期）
- 个人角色怎么定（AI + 后端、不夸大团队与年限）
- 薪资与跳槽怎么对齐（成都、30w、远程、非外包）
- 如何把同一套逻辑迁到别的业务（问数、小程序、其他 Agent）

## 二、核心叙事链条（按推理顺序）

业务场景 → 能力边界（一期）→ 架构分流 → 技术选型理由 → 二期演进 → 个人分工与时间线 → 面试话术 → 风险与诚实边界

每条结论都要能回答「为什么不是另一种做法」。（例：有 KG 仍要 ES，不是再堆 RAG；二期不是没活，是活从模型转向工程与运营。）

## 三、架构叙事逻辑（医疗导诊 → 可抽象）

| 原则 | 本会话结论 | 抽象到其他项目 |
|------|-----------|---------------|
| 分流 | 症状/科室走 KG；挂号/报销/位置走 ES+规则 | 结构化推理 与 文档/流程检索 分开 |
| 输出形态 | 科室可只要 JSON，不必长 LLM 生成 | 结构化结果 与 自然语言说明 分开 |
| RAG 角色 | 别名/否定归一可轻量 RAG；不全文融合再生成 | RAG 用于 归一/召回，不替代主决策 |
| 传统开发 | 指南、预约、API 以规则+后端为主 | AI 做理解与路由，业务做执行 |
| LoRA | 非主线；意图/槽位数据够才考虑 | 优先 评测+规则+知识库，后考虑微调 |
| 评测 | 无 Golden/指标 = 面试弱于「能部署」 | 任何 Agent 项目都应有 离线集+指标 |

**关键词：** 双通道、护栏、槽位、MCP/Tool、准上线、Golden Set。

## 四、一期 / 二期叙事逻辑

| 维度 | 一期（必须讲清） | 二期（避免「没活」） |
|------|-----------------|---------------------|
| 目标 | 核心能力稳定 + 安全 | 办事闭环、规模化、运营 |
| 医疗例 | 科室推荐稳定、对话护栏 | 预约/HIS、多院区、评测回归、后台 |
| AI 占比 | 高（编排、KG、意图） | 中（路由、Tool、监控）；后端对接升高 |
| 时间 | 约 6～12 月到试点 | 第二年 = 对接+运营+扩展，不是重复造 KG |

**口述公式：**

「第一年把分科和安全带稳；第二年把能力和业务系统、评测运营接起来。」

## 五、团队与个人角色叙事逻辑

| 角色 | 职责边界（面试用） |
|------|-------------------|
| 你（AI + 后端） | LangGraph/意图/槽位/KG/评测 + 导诊域 API、会话、MCP 联调 |
| 业务后端 | 挂号、缴费、HIS、支付（协作，不抢功） |
| 前端 | 小程序页、对话 UI |
| 产品 | PRD、Golden 验收、bad case 优先级 |

**禁止：** 「四人团队全职 2 年做完整个 V3.0 所有图标」——除非真能逐项答出。

**推荐：** 「平台多模块；我主责智能导诊 AI + 导诊服务 API，深度约 X 月/年。」

**时间叙事：**

- 项目存续 2 年 ≠ 你 24 个月都在写 LangGraph
- 拆成：核心建设 N 月 + 运营/对接 M 月
- 联想半年 AI 单独说，不与导诊 2 年混成「职业生涯只有 AI 半年」

## 六、薪资与求职叙事逻辑（成都 · 30w · 远程 · 非外包）

| 断言 | 逻辑 |
|------|------|
| 30w 远程 | 可冲，靠 作品+指标+角色清晰，不靠虚报 2 年纯 AI |
| 评测必须 | 无指标 ≈ 只能谈 22～26w 档 |
| 真上线非必须 | Docker/staging + 录屏 + eval.md = 准上线 |
| 后端+AI 2 年 | 比纯 AI 2 年更可信（第二年对接 API、评测） |
| 开源 Agent1 仓 | 作 架构参考；面试讲 线上医速递+个人职责，不拿 3 commit 冒充 2 年工时 |

## 七、面试问答叙事模板（可直接套）

| 问题 | 叙事要点 |
|------|---------|
| Skills？ | FastAPI、LangGraph、Redis、Neo4j/ES、MCP、DashScope、Docker、评测脚本 |
| 多 Agent？ | LangGraph 多节点分工，非多个自治 Bot；科室 只信 KG |
| 其他业务？ | 联想问数/RAG（脱敏）+ 医疗导诊垂直 |
| 做了多久？ | 项目 X 月/年；个人深度 Y 月；分阶段讲 |
| KG 为何还要 RAG？ | KG 算科室；RAG/ES 做指南与术语归一；长生成可砍 |
| 二期和 AI 有关吗？ | 有关，但是 Tool、路由、评测；不是再训大模型 |
| 后端够面试吗？ | AI 应用岗够；纯资深后端需补鉴权/HIS/体量 |

## 八、与其他项目的类比映射（延续问询时用）

在本会话里建立的 **类比关系**，换项目时可继续用：

| 医疗导诊 | 可类比到 |
|---------|---------|
| 科室 KG | 问数 字段/标准枚举、风控 规则集 |
| ES 指南 | 企业 制度/FAQ/流程文档 |
| 槽位问诊 | 多轮澄清、必填参数 |
| 心痛/心疼 | 缩写 vs 全称、歧义词澄清 |
| kpi + describe JSON | key_word + describe 召回优于长串拼接 |
| 护栏/危急 | 合规拒答、敏感操作拦截 |
| 医速递小程序 | 任何 前端 + 中台 + AI 子服务 形态 |

**延续问询句式：**

「按 cursor-session-narrative 里的双通道逻辑，我这个 XX 项目应该怎么拆一期二期？」

## 九、诚实边界（叙事红线）

回答或写简历时 **主动遵守**，避免被追问穿帮：

- 不虚构 四人全职 2 年、不虚构 全院正式上线（无证据时）。
- 科室/字段 不交给 LLM 自由生成，来自 KG/库/规则。
- 外包半年 与 导诊 2 年 分开陈述。
- 开源仓 3 commit 不说成「两年唯一工作成果」。
- 没有评测数字时，不说「效果已经很好」。

## 十、在其他 Cursor 项目中怎么用

**新开 Chat 第一句建议：**

```
请按 @docs/cursor-session-narrative.md 的叙事逻辑回答：
- 先分流（核心 AI 能力 vs 传统/检索）
- 再一期/二期
- 再我的角色（AI+后端）与时间线
- 标明诚实边界
我的新问题是：……
```

- 若只问技术、不问面试：「只用第三节架构原则，不用薪资段落。」
- 若问问数/联想：「用第八节类比表，把科室 KG 换成字段映射。」

## 十二、一句话总逻辑

先定业务边界与输出形态 → 再双通道分流（推理 vs 检索/规则）→ 一期稳核心+护栏+评测 → 二期接业务与运营 → 个人用 AI+后端+真实工期陈述 → 所有说法可追问到指标与职责，不夸大。
